# 数据分析业务流程

## 流程目标

描述 CPV 系统中三类数据分析入口的业务流程：数据分析工作台、监控项目模块和相关性分析汇总模块。测试用例生成时，应重点区分入口来源、可选分析方法范围、数据来源、数据处理、分析配置、结果生成以及结果处理差异。

## 业务流程图

```
数据分析入口
       │
       ▼
选择入口类型
   ├─ 数据分析工作台：自由选择一个或多个分析方法
   ├─ 监控项目模块：仅当前监控项目配置的方法可分析
   └─ 相关性分析汇总模块：仅相关性帕累托排序图、相关性分析可分析
       │
       ▼
选择数据源 / 可选数据处理
       │
       ▼
进入数据分析详情页
       │
       ▼
完成动态表单配置
       │
       ▼
开始分析
       │
       ▼
生成分析结果
       │
       ▼
结果处理
   ├─ 数据分析工作台：分析完成后自动保存分析结果
   ├─ 监控项目模块：将分析结果推送至CPV系统
   └─ 相关性分析汇总模块：将分析结果推送至CPV系统
```

## 通用流程

三类入口进入数据分析详情页后的基础能力一致，差异主要体现在分析方法选择范围和结果处理方式。

1. 用户选择或进入目标数据分析入口。
2. 用户选择数据源，数据来源支持中台数据对接同步、MES 集成同步和本地上传数据。
3. 用户可跳过数据处理直接分析，也可先执行筛选、排序、转换和剔除。
4. 用户进入数据分析详情页，根据入口规则选择或查看当前分析方法。
5. 用户完成当前分析方法对应的动态表单配置。
6. 点击开始分析前，系统校验数据源、配置完整性、变量类型、字段匹配和动态表单必填项。
7. 系统生成分析结果，并按当前分析方法展示适用结果区域。
8. 用户可查看历史结果，并按入口能力继续执行数据替换、数据处理或一键分析。

## 分析方法范围

为避免多处维护同一长列表，本文件只维护入口范围规则，不在每个流程中重复列出全部分析方法。

| 范围类型 | 分析方法 |
|---|---|
| 相关性分析方法 | 相关性分析、相关性帕累托排序图 |
| 监控类分析方法 | 正态性检验、汇总报告图、单值控制图、多个Y值的单值图、单值移动极差控制图、百分位数控制图、P控制图、NP控制图、Xbar-S 控制图、Xbar-R 控制图、持续监控图、过程能力分析（正态）、过程能力分析（非正态）、帕累托图、拟合线图、箱线图、趋势图、单折线图、多折线图、双样本 T 检验、配对T检验、双样本等价检验、单因子方差分析 |
| 全部分析方法 | 监控类分析方法 + 相关性分析方法 |

## 三类入口差异

| 分析入口 | 入口来源 | 分析方法选择规则 | 可分析范围 | 结果处理 |
|---|---|---|---|---|
| 数据分析工作台 | 数据分析工作台的数据分析入口 | 用户可自由选择一个或多个分析方法 | 全部分析方法 | 分析完成后自动保存分析结果，可继续数据替换、数据处理和一键分析 |
| 监控项目模块 | 监控项目列表的数据分析入口 | 展示全部分析方法入口，仅当前监控项目配置的分析方法高亮，其他置灰不可切换 | 监控类分析方法，不支持相关性分析类方法 | 将分析结果推送至CPV系统，更新监控项目分析状态和分析结果名称 |
| 相关性分析汇总模块 | 相关性分析汇总列表的数据分析入口 | 展示全部分析方法入口，仅当前相关性分析汇总配置的分析方法高亮，其他置灰不可切换 | 仅支持相关性帕累托排序图、相关性分析 | 将分析结果推送至CPV系统，更新相关性分析汇总状态和分析结果名称 |

## 结果区域口径

分析结果区域必须以当前分析方法的 PRD / UI 为准，不同方法的结果结构可以不同。

- 不得统一假设所有方法都有分析图、过程数据、系统结论、输入结论或用户自定义结论。
- 若当前方法没有某个结果区域，不得强行生成对应展示或校验用例。
- 生成结果页用例时，应先从 PRD / UI 提取当前方法实际存在的结果区域、字段、图表、表格、结论和操作入口。
- 涉及将分析结果推送至CPV系统时，只验证当前方法实际支持推送的数据、结果内容、结论信息和推送状态。

## 关键业务规则

- 三类入口的数据来源能力一致，均支持中台数据对接同步、MES 集成同步和本地上传数据。
- 数据处理不是必选步骤；未执行数据处理时应使用原始数据进行分析，执行数据处理后应使用处理后的数据进行分析。
- 数据分析工作台可自由选择全部分析方法，分析完成后自动保存分析结果。
- 监控项目模块只能执行当前监控项目配置的监控类分析方法，不得生成相关性分析和相关性帕累托排序图执行用例。
- 相关性分析汇总模块只能执行相关性帕累托排序图和相关性分析，不得生成箱线图、趋势图、控制图、过程能力分析等监控类分析方法执行用例。
- 监控项目模块和相关性分析汇总模块可展示全部分析方法入口，但仅当前配置的分析方法可操作，其他方法置灰且不可切换。
- 点击开始分析前，必须校验数据源、动态表单配置、变量类型和字段匹配。
- 监控项目模块和相关性分析汇总模块将分析结果推送至CPV系统后，应记录推送状态，并更新对应分析状态和分析结果名称。

## 测试重点

- 三类入口是否进入正确的数据分析详情页，且页面来源和上下文正确。
- 中台数据、MES 集成数据、本地上传数据在三类入口下是否都可选择并正确预览。
- 未执行数据处理时，分析是否使用原始数据；执行筛选、排序、转换和剔除后，分析是否使用处理后的数据。
- 数据分析工作台是否支持全部分析方法自由选择。
- 数据分析工作台分析完成后是否自动保存分析结果，历史结果是否可查看。
- 监控项目模块是否仅允许分析当前监控项目配置的分析方法，相关性帕累托排序图和相关性分析是否被排除。
- 相关性分析汇总模块是否仅允许相关性帕累托排序图和相关性分析，其他监控类分析方法是否被排除。
- 置灰分析方法是否不可点击、不可切换、不可配置、不可开始分析。
- 动态表单配置是否随分析方法正确联动。
- 必填配置缺失时是否禁止开始分析并给出明确提示。
- 分析结果是否按当前分析方法展示适用区域，且结果内容与数据源、配置参数和数据处理规则一致。
- 数据替换或数据处理后，一键分析是否按新数据重新计算，并正确展示成功、部分失败和全部失败提示。
- 监控项目模块或相关性分析汇总模块将分析结果推送至CPV系统后，是否正确记录推送状态，并更新对应分析状态和分析结果名称。
