# 一键分析业务流程

## 流程目标

描述数据分析详情页中“数据替换”和“数据处理”触发一键分析的核心流程。用户可在不退出分析流程的情况下更换底层数据源或调整数据处理规则，系统根据当前已使用分析方法生成的结果，尝试基于变更后的数据批量重新生成分析结果。测试用例生成时，应重点关注入口范围、触发方式、字段匹配、变量类型与数据分布校验、结果提示、未分析原因和异常恢复。

## 业务流程图

```
数据分析详情页
       │
       ▼
触发一键分析
   ├─ 数据替换：选择新数据源并确认
   └─ 数据处理：进行数据处理后返回详情页自动触发
       │
       ▼
遍历当前已使用分析方法生成的结果
       │
       ▼
按字段名称匹配变更后的数据字段
       │
       ▼
校验变量类型和数据分布
       │
       ▼
执行一键分析
       │
       ▼
展示执行结果
   ├─ 全部成功：查看已生成分析结果
   ├─ 部分成功部分失败：查看成功结果和未分析原因
   └─ 全部失败：确认清空配置或取消保留当前状态
       │
       ▼
用户查看结果，或修复未分析项后重新分析
```

## 适用入口

一键分析可在以下数据分析详情页触发：

1. 数据分析工作台模块数据分析详情页。
2. 监控项目模块数据分析详情页。
3. 相关性分析汇总模块数据分析详情页。

## 触发方式

### 数据替换触发

1. 用户在数据分析详情页点击数据替换。
2. 用户选择新数据源，可选择中台数据对接同步表、MES 集成同步表、本地上传数据表或数据处理后的表。
3. 用户确认替换后，系统自动触发一键分析。

### 数据处理触发

1. 用户在数据分析详情页点击数据处理。
2. 用户对当前数据执行筛选、排序、转换或剔除。
3. 用户返回数据分析详情页，并基于处理后的数据自动触发一键分析。

## 主流程

1. 数据替换或数据处理完成后，系统遍历当前已使用分析方法生成的结果。
2. 系统按字段名称匹配变更后的数据字段。
3. 系统校验变量类型和数据分布是否满足对应分析方法要求。
4. 系统对可重新分析的分析方法执行一键分析并批量重新生成分析结果。
5. 系统根据执行结果展示全部成功、部分成功部分失败或全部失败提示。
6. 用户确认提示后查看分析结果；未分析项保留在页面中，供用户手动修复配置后重新分析。

## 结果提示

### 全部成功

1. 所有已使用分析方法均成功重新生成分析结果。
2. 系统提示：`当前分析包含 [总数] 个子分析结果，已完成快速分析 [成功数] 个。`
3. 用户确认后查看重新生成的分析结果。

### 部分成功部分失败

1. 部分已使用分析方法重新生成结果成功，部分分析方法未能分析。
2. 系统提示：`当前分析包含 [总数] 个子分析结果，已完成快速分析 [成功数] 个，未分析 [失败数] 个。未分析原因请至页面查看。`
3. 用户确认后进入配置并分析页面。
4. 未分析项显示“未分析”状态标签。
5. 鼠标悬停“未分析”标签时，系统展示具体未分析原因。

### 全部分析失败

1. 所有已使用分析方法都无法基于变更后的数据重新生成分析结果。
2. 系统弹窗提示：`当前分析项与数据源不匹配，或全部数据分析失败。系统将清空您已设置的参数，是否继续？`
3. 用户确认继续后，系统清空已设置参数。
4. 用户取消时，系统保持当前页面状态，不继续清空配置。

## 一键分析失败原因

以下原因命中后，对应分析结果状态变为“未分析”，页面保留该未分析项，用户可查看原因。若失效字段仍保留在配置中，可通过数据替换或数据处理修复后重新参与一键分析；若字段已被清空，则需手动重新配置后再开始分析，不归入一键分析修复成功场景。

### 定类变量导致未分析

1. 系统发现当前分析方法依赖字段在变更后的数据中匹配上定类变量，或字段变为非数值 / 全空分析列。
2. 鼠标悬停提示：`XXX 变量为定类变量，暂不支持分析。`
3. 系统清空配置区域中涉及的失效字段，不显示 `NULL`。

### 字段删减导致未分析

1. 系统发现当前分析方法依赖字段在变更后的数据中不存在。
2. 鼠标悬停提示：`XX、XXX 字段有删减，请手动分析。`
3. 系统清空配置区域中涉及的失效字段，不显示 `NULL`。

### 正态性变化导致未分析

1. 系统发现数据替换或数据处理导致字段正态性发生变化。
2. 当前仅过程能力分析（正态）和过程能力分析（非正态）存在该类场景。
3. 鼠标悬停提示：`XXX 正态性发生改变，请手动分析。`

### 阶段内容不一致导致未分析

1. 系统发现当前分析方法依赖的阶段字段在变更后的数据中不存在，或已配置参考线的阶段在变更后的数据中不存在。
2. 鼠标悬停提示：`用此变量定义阶段下的内容不完全相同，暂不支持分析。`

### 其他原因导致未分析

1. 系统发现数据替换或数据处理后，当前分析方法触发单项分析错误。
2. 鼠标悬停提示沿用点击开始分析时报错逻辑，例如“分析列中存在无效数值”“没有足够的数据”“阶段数超过 5 个”等。

## 未分析原因优先级

| 优先级 | 原因类型 | 提示文案 |
|---|---|---|
| 1 | 定类变量暂不支持分析 | `XXX 变量为定类变量，暂不支持分析。` |
| 2 | 字段有删减 | `XXX、XXX 字段有删减，请手动分析。` |
| 3 | 正态性发生变化 | `XXX 正态性发生改变，请手动分析。` |
| 4 | 用此变量定义阶段下的内容不完全相同 | `用此变量定义阶段下的内容不完全相同，暂不支持分析。` |
| 5 | 其他原因分析错误 | 沿用点击开始分析时报错逻辑，例如“分析列中存在无效数值”“没有足够的数据”“阶段数超过 5 个”等。 |

## 异常场景

### 数据源失效

1. 用户用于分析的数据表被停用或失效。
2. 进入数据分析详情页时提示：`当前数据表状态已失效，暂不支持分析。`
3. 用户需重新选择有效数据源后才能继续分析。

### 网络或服务器异常

1. 系统一键分析过程中发生网络中断或服务器异常。
2. 系统中断本次一键分析并提示失败原因。
3. 页面应保留可恢复状态，避免配置丢失且无提示。

## 关键业务规则

- 一键分析仅在数据替换或数据处理后自动触发。
- 字段匹配优先按字段名称匹配，再校验变量类型和数据分布要求。
- 每个未分析项只展示命中的最高优先级原因。
- 定类变量、字段删减、字段缺失或字段匹配失败导致配置字段失效时，配置区域应清空失效字段，不显示 `NULL`。
- 全部失败且用户取消时，不应清空已有配置或分析结果。
- 失效字段仍保留时，通过数据替换或数据处理修复并一键分析成功后，应清除“未分析”状态；字段已被清空时，需要手动重新配置后再开始分析。

## 测试重点

- 三类入口是否都能通过数据替换和数据处理触发一键分析。
- 数据替换是否支持中台数据对接同步表、MES 集成同步表、本地上传数据表和数据处理后的表。
- 数据处理后的筛选、排序、转换、剔除结果是否参与一键分析。
- 字段名称、变量类型和数据分布匹配是否准确。
- 全部成功、部分成功部分失败、全部失败三类提示是否准确。
- 全部失败时确认和取消的处理结果是否正确。
- 未分析标签、悬停原因和原因优先级是否正确。
- 字段删减、字段缺失、字段匹配失败或匹配上定类变量导致配置失效时，配置区域是否清空且不显示 `NULL`。
- 数据源失效、网络中断和服务器异常时是否有明确提示并保留可恢复状态。
