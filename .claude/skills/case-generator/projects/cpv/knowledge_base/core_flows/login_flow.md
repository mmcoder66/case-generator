# 登录与站点选择流程

## 流程目标

描述用户从登录页输入账号密码、按需选择可访问站点、完成账号状态和账号密码校验，到跳转至公共管理系统统一门户、公共管理站点或自定义业务站点的完整流程。测试用例生成时，应重点关注账号状态校验、账号密码校验、站点列表过滤、未选择站点与选择不同站点的跳转差异、登录状态记录和异常登录场景。

## 业务流程图

```
进入登录页
       │
       ▼
输入账号密码
       │
       ▼
加载可访问站点列表
   ├─ 无权限站点：不展示
   └─ 停用站点：不展示
       │
       ▼
选择目标站点或不选择站点
       │
       ▼
提交登录
       │
       ▼
账号状态与账号密码校验
   ├─ 用户停用：拒绝登录，停留登录页
   ├─ 用户失效：拒绝登录，停留登录页
   ├─ 用户锁定：拒绝登录，停留登录页
   ├─ 账号密码错误：拒绝登录，停留登录页
   └─ 校验通过：记录登录状态和用户信息
       │
       ▼
登录成功去向
   ├─ 不选择站点：进入公共管理系统统一门户，提供可访问站点选择
   ├─ 公共管理站点：进入站点管理页
   └─ 自定义业务站点：进入对应站点首页
```

## 主流程

1. 用户进入系统登录页。
2. 用户输入账号和密码。
3. 系统根据用户账号查询可访问站点，并在登录页展示站点下拉列表。
4. 用户可从站点下拉列表中选择目标站点，也可不选择站点直接提交登录。
5. 系统验证账号状态和账号密码。
6. 校验通过后，系统记录登录状态、用户信息和可访问站点列表；若用户选择站点，同时记录当前站点 ID 和站点类型。
7. 系统根据是否选择站点及所选站点类型跳转至对应页面。

## 登录成功去向

### 不选择站点

1. 状态正常的用户输入正确账号密码。
2. 用户不选择站点，直接提交登录。
3. 系统完成账号状态和账号密码校验。
4. 登录成功后进入公共管理系统统一门户。
5. 统一门户提供当前用户可访问站点供用户选择；若用户无任何可访问站点，则站点列表为空或不可选。

### 选择公共管理站点

1. 用户账号拥有公共管理访问权限。
2. 用户在站点下拉列表中选择“公共管理”。
3. 系统完成账号状态和账号密码校验。
4. 登录成功后默认进入站点管理页。

### 选择自定义业务站点

1. 用户账号拥有某自定义业务站点访问权限。
2. 用户在站点下拉列表中选择该自定义站点。
3. 系统完成账号状态和账号密码校验。
4. 登录成功后跳转至该站点首页。
5. 当前站点在页面顶部站点列表中高亮展示。

## 异常分支

### 用户停用

1. 用户输入已停用账号和密码。
2. 用户选择目标站点或不选择站点，并提交登录。
3. 系统校验用户状态为停用。
4. 系统拒绝登录，并提示用户不可登录。
5. 用户停留在登录页，不记录登录状态。

### 用户失效

1. 用户输入已失效账号和密码。
2. 用户选择目标站点或不选择站点，并提交登录。
3. 系统校验用户状态为失效。
4. 系统拒绝登录，并提示用户不可登录。
5. 用户停留在登录页，不记录登录状态。

### 用户锁定

1. 用户输入已锁定账号和密码。
2. 用户选择目标站点或不选择站点，并提交登录。
3. 系统校验用户状态为锁定。
4. 系统拒绝登录，并提示用户不可登录。
5. 用户停留在登录页，不记录登录状态。

### 账号密码错误

1. 用户输入账号和密码。
2. 用户选择目标站点或不选择站点，并提交登录。
3. 系统校验账号密码失败。
4. 系统拒绝登录，并提示账号或密码错误。
5. 用户停留在登录页，可重新输入账号密码后再次登录。

### 无权限站点不展示

1. 用户输入账号。
2. 系统查询该用户可访问站点列表。
3. 用户无访问权限的站点不展示在站点下拉列表中。
4. 用户无法选择无权限站点，也不能通过登录页进入该站点。

### 用户无任何可访问站点

1. 用户输入账号。
2. 系统查询该用户可访问站点列表。
3. 如果用户无任何可访问站点，站点下拉列表为空或不可选。
4. 用户仍可不选择站点并提交登录。
5. 登录成功后进入公共管理系统统一门户，统一门户站点列表为空或不可选。

### 停用站点不展示

1. 用户输入账号。
2. 系统查询该用户可访问站点列表。
3. 已停用站点不展示在站点下拉列表中。
4. 用户无法选择停用站点，也不能通过登录页进入该站点。

## 业务规则

1. 用户登录必须通过账号状态和账号密码校验。
2. 站点下拉列表只展示当前用户可访问站点，包括公共管理站点和自定义业务站点。
3. 公共管理站点与自定义业务站点共用统一认证体系，登录成功后无需在站点间重复登录。
4. 登录成功后的登录状态记录应包含用户标识、用户名、登录状态和可访问站点列表；选择站点登录时还应包含当前站点 ID 和站点类型。
5. 不选择站点时进入公共管理系统统一门户，并提供当前用户可访问站点进行选择；若用户无任何可访问站点，仍可进入统一门户，但站点列表为空或不可选。
6. 选择公共管理站点时默认进入站点管理页；选择自定义业务站点时进入对应业务站点首页。
7. 无权限站点和停用站点不展示在站点下拉列表中，不可被登录访问，其关联数据在业务中不可见或不可操作。
8. 用户停用、失效或锁定时不可登录任何站点，且不得记录登录状态。

## 测试关注点

1. 正常用户不选择站点且账号校验通过时，是否进入公共管理系统统一门户并展示当前用户可访问站点。
2. 正常用户选择公共管理站点登录时，是否默认进入站点管理页。
3. 正常用户选择自定义业务站点登录时，是否进入对应站点首页。
4. 站点下拉列表仅展示当前用户可访问站点。
5. 用户停用、失效或锁定时不得登录任何站点。
6. 账号密码错误时不得登录任何站点。
7. 用户无权限访问的站点不应出现在站点下拉列表中。
8. 用户无任何可访问站点时，是否仍可不选择站点登录并进入公共管理系统统一门户。
9. 停用站点不应出现在站点下拉列表中。
10. 登录成功后记录的用户信息、登录状态、当前站点信息与实际选择一致。
11. 公共管理站点和自定义业务站点之间跳转时不应要求重复登录。
12. 选择站点登录后，当前所在站点应在站点列表中高亮展示。
