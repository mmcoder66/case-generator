# CPV 主业务流程

## 流程目标

描述 CPV（持续工艺确认）系统从年度计划、任务、方案模板、方案编制、相关性分析汇总、监控项目到报告编制的核心业务链路。测试用例生成时，应重点关注跨模块数据带入、状态流转、审批控制、版本管理、分析结果引用、数据分析工作台结果引用和报告输出一致性。

## 业务流程图

```

       │
       ▼

   ├─ 年度计划生效后生成周期性任务
   └─ 手动创建任务
       │
       ▼
任务管理跟踪任务执行
       │
       ▼
方案编制创建 CPV 方案
   ├─ 任务管理：选择已生效任务
   ├─ 方案模板：提供已生效模板结构
   └─ 方案内容：添加相关性分析项目
       │
       ▼
方案创建完成
   ├─ 相关性分析项目在相关性分析汇总菜单下形成分析项目，并可进入数据分析
   │   └─ 完成数据分析后推送至业务系统
   └─ 方案提交审批，审批通过后生效
       │
       ▼

   ├─ 通过已分析或已确认的相关性分析项目创建监控项目
   └─ 在监控项目模块新增监控项目
       │
       ▼

       │
       ▼
报告编制生成 CPV 报告
   ├─ 报告模板：提供已生效报告结构
   ├─ 关联方案：根据所选任务自动填充方案
   ├─ 监控项目：根据关联方案自动填充监控项目
   └─ 报告内容：插入相关性分析项目结果、监控项目结果、数据分析工作台结果等数据

       │
       ▼
```

## 核心业务链路

1. 年度计划管理中创建 CPV 年度计划；计划审批生效后，可按计划规则生成周期性任务。
2. 任务也可在任务管理中手动创建，任务管理负责跟踪待执行、进行中、完成、关闭等状态。
3. 方案模板提供标准方案结构；方案编制选择已生效任务和已生效方案模板创建具体 CPV 方案，并在方案中添加相关性分析项目。
4. 方案创建后，方案中的相关性分析项目会在相关性分析汇总菜单下形成分析项目，并可进入数据分析；完成分析后进入已分析状态，可推送至业务系统并确认结果。
5. 方案提交审批并通过后生效，生效方案作为创建正式监控项目和报告编制关联方案的依据。
6. 监控项目可通过已分析或已确认的相关性分析项目创建，也可在监控项目模块手动新增。
7. 监控项目完成数据分析后，需推送至业务系统，并确认监控项目结果和相关性分析项目结果。
8. 报告编制选择任务后自动带出关联方案，并根据关联方案自动填充监控项目；结合已生效报告模板，插入相关性分析项目结果、监控项目结果、数据分析工作台结果等数据生成 CPV 报告。
9. 报告审批生效后，推进任务状态并锁定关键报告内容，形成业务闭环。

## 模块状态流转

| 模块 | 状态流转 |
|---|---|
| 年度计划管理 | 草稿 -> 审批 -> 已生效 -> 失效；支持退回、撤回、变更 |
| 任务管理 | 草稿 -> 审批 -> 待执行 -> 进行中 -> 完成 -> 关闭；支持退回、撤回、升版 |
| 方案模板 | 草稿 -> 已生效 -> 失效；支持升版 |
| 方案编制 | 草稿 -> 审批 -> 已生效 -> 失效；支持退回、撤回、升版 |
| 监控项目 | 草稿 -> 未分析 -> 已分析 -> 已确认 -> 关闭；支持修订 |
| 相关性分析汇总 | 草稿 -> 未分析 -> 已分析 -> 已确认 -> 关闭；支持变更 |
| 报告模板 | 草稿 -> 已生效 -> 失效；支持升版 |
| 报告编制 | 草稿 -> 审批 -> 已生效 -> 失效；支持退回、撤回、升版 |

## 模块关系

| 起点模块 | 关系 | 目标模块 | 说明 |
|---|---|---|---|
| 年度计划管理 | 生成 | 任务管理 | 年度计划生效后，根据计划规则生成周期性任务。 |
| 任务管理 | 创建 | 任务管理 | 支持手动创建任务。 |
| 任务管理 | 关联 | 方案编制 | 方案编制选择已生效任务创建具体 CPV 方案，任务用于跟踪执行和方案状态。 |
| 方案模板 | 引用 | 方案编制 | 方案编制引用已生效的方案模板创建标准化方案。 |
| 方案编制 | 添加 | 相关性分析项目 | 方案创建时可添加相关性分析项目；方案创建后，该项目作为相关性分析汇总菜单下的分析项目，可进入数据分析。 |
| 相关性分析汇总 | 创建依据 | 监控项目 | 相关性分析汇总中已分析或已确认的相关性分析项目可用于创建监控项目。 |
| 监控项目 | 创建 | 监控项目 | 支持在监控项目模块手动新增监控项目。 |
| 任务管理 | 关联 | 报告编制 | 报告创建时选择任务，系统根据所选任务自动填充关联方案。 |
| 方案编制 | 关联 | 报告编制 | 关联方案为报告自动填充监控项目提供范围。 |
| 监控项目 | 引用 | 报告编制 | 监控项目结果确认后，报告编制才可插入已确认的结果。 |
| 相关性分析汇总 | 引用 | 报告编制 | 相关性分析项目结果确认后，报告编制才可插入已确认的结果。 |
| 数据分析工作台 | 引用 | 报告编制 | 数据分析工作台已分析的结果可作为报告内容插入报告。 |
| 报告模板 | 引用 | 报告编制 | 报告编制引用已生效的报告模板生成报告结构。 |

## 关键业务规则

- 年度计划未生效前，不应生成正式周期性任务；任务也可在任务管理中手动创建。
- 未生效任务不应被方案编制选择用于创建 CPV 方案。
- 已失效的年度计划、方案模板或报告模板不应被新任务、方案或报告引用。
- 方案创建前，相关性分析项目不应在相关性分析汇总菜单下形成分析项目，也不应进入数据分析。
- 未生效方案不应作为创建正式监控项目或报告编制关联方案的依据。
- 通过相关性分析项目创建正式监控项目时，相关性分析项目应已完成数据分析。
- 监控项目可通过已分析或已确认的相关性分析项目创建，也可在监控项目模块手动新增。
- 相关性分析项目未推送至业务系统或未确认结果前，不应作为最终报告结论依据。
- 报告创建时应先选择任务，并根据所选任务自动带出关联方案、根据关联方案自动填充监控项目。
- 监控项目未完成数据分析、未推送至业务系统或未确认结果前，不应作为最终报告结论依据。
- 报告编制应基于已确认的相关性分析项目结果、已确认的监控项目结果和数据分析工作台已分析结果。
- 审批退回后，记录应回到可调整状态，并保留退回原因。
- 升版应保留历史版本，已生效历史版本不应被覆盖。
- 报告生效后，应推进相关任务状态，并锁定关键报告内容。

## 测试重点

- 各模块状态流转是否符合审批和生效规则。
- 年度计划生效后是否正确生成周期性任务，任务管理是否支持手动创建任务。
- 方案编制创建方案时是否只能选择已生效任务和已生效方案模板。
- 任务、方案、相关性分析项目、监控项目、报告之间的关联是否完整且可追溯。
- 模板失效或升版后，历史方案和报告是否保持版本一致性。
- 方案创建时是否可添加相关性分析项目，方案创建后是否在相关性分析汇总菜单下形成分析项目并可进入数据分析。
- 方案审批通过后是否正确生效，未生效方案是否不能作为正式监控项目和报告编制关联方案的依据。
- 相关性分析项目达到已分析或已确认状态后，是否可创建监控项目。
- 报告创建时选择任务后是否自动填充关联方案，并根据关联方案自动填充监控项目。
- 相关性分析项目结果、监控项目结果和数据分析工作台结果是否能被报告正确插入和引用。
- 报告生效后任务状态是否正确推进。
- 退回、撤回、变更、升版、失效等分支流程是否保留操作原因和审计追踪。
