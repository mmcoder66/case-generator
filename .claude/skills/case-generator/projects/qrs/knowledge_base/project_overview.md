## 一、系统定位

**系统名称**：质量回顾系统（Quality Review System，简称 **QRS**；文档中也称 **PQRS** — Product Quality Review System）

**系统定位**：质量回顾系统是支撑制药企业 **持续工艺确认（Continued Process Verification，CPV）** 与 **产品质量回顾（Product Quality Review，PQR）** 的核心业务管理平台。系统以 **ICH Q8/Q9/Q10** 质量风险管理框架和 **GMP 法规要求**为设计基准，覆盖从回顾计划制定、任务分解与派发、原始数据收集与多级复核、统计分析计算到质量回顾报告生成与审批放行的 **全生命周期管理**，实现质量回顾业务的 **数字化、标准化、合规化与可追溯**。

**法规与设计基准**：
- ICH Q8(R2) / Q9 / Q10 质量风险管理框架
- 《药品生产质量管理规范（2010 年修订）》（GMP）
- EU GMP Annex 15
- FDA Process Validation 指南
- **ALCOA+** 数据完整性原则（Attributable、Legible、Contemporaneous、Original、Accurate + Complete、Consistent、Enduring、Available）

---

## 二、核心价值主张

| 价值维度 | 具体描述 |
|---|---|
| **流程标准化** | 将质量回顾的业务流程固化为系统标准工作流，消除人工操作中的随意性与遗漏风险 |
| **数据可追溯** | 所有数据操作（录入、修改、复核、审批）均记录完整的审计追踪日志，满足法规审计要求 |
| **分析自动化** | 预置多种统计分析模版，实现一键式数据分析与图表生成，减少人工计算错误 |
| **协作高效化** | 通过任务管理与待办机制，实现跨部门（QA / 生产 / 实验室 / 工艺）的高效协同 |
| **合规保障化** | 内置数据完整性控制（ALCOA+ 原则）、审批工作流、电子签名等合规机制 |

## 三、系统功能架构图

PQRS 系统由 **11 个核心业务功能模块**组成，按业务层次划分为 **六层**。

| 层次 | 包含模块 | 层次职责 |
|---|---|---|
| **门户展现层** | 首页 | 提供统一的工作入口与业务态势感知 |
| **计划管理层** | 回顾计划、任务管理 | 计划制定、审批、分解与任务跟踪 |
| **数据管理层** | 数据表模版、数据复核 | 数据结构的定义与原始数据的全生命周期管理 |
| **分析管理层** | 数据分析、分析模版、分析结果 | 统计方法的配置、分析执行与结果管理 |
| **报告管理层** | 报告模版、回顾报告 | 报告结构的定义、报告编写与审批 |
| **基础数据层** | 回顾对象 | 管理产品、设备、方法、供应商等基础数据及其 CQA/CPP/IPC 参数 |

### 架构分层图示

```text
┌─────────────────────────────────────────────────────────────┐
│  门户展现层 │                 首页                          │
├─────────────────────────────────────────────────────────────┤
│  计划管理层 │          回顾计划 │ 任务管理                   │
├─────────────────────────────────────────────────────────────┤
│  数据管理层 │       数据表模版 │ 数据复核                    │
├─────────────────────────────────────────────────────────────┤
│  分析管理层 │   数据分析 │ 分析模版 │ 分析结果                │
├─────────────────────────────────────────────────────────────┤
│  报告管理层 │          报告模版 │ 回顾报告                   │
├─────────────────────────────────────────────────────────────┤
│  基础数据层 │                 回顾对象                       │
└─────────────────────────────────────────────────────────────┘
```

**关键依赖原则**：
- **基础数据层是全部上层模块的事实基座**，回顾对象一旦变更将级联影响所有上层业务。
- **数据流自下而上单向贯通**：基础数据 → 原始数据 → 分析结果 → 报告输出。
- **任务管理与首页是横切模块**：任务管理贯穿计划→执行全链路；首页聚合所有模块的统计与待办。

## 四、核心业务主线流程

> 主线：**回顾对象 → 数据表模版 / 分析模版 → 数据复核 → 数据分析 → 分析结果 → 报告模版 → 回顾报告**，由 **回顾计划 → 任务管理** 驱动执行，由 **首页** 聚合呈现。

### 4.1 模块依赖关系矩阵

| 模块名称 | 依赖上游模块 | 被下游模块依赖 | 数据传递内容 | 触发时机 |
|---|---|---|---|---|
| **回顾对象** | 无（基础数据层） | 全部其他 10 个模块 | 对象基本信息、CQA/CPP/IPC 参数定义、规格限值 | 实时引用 |
| 数据表模版 | 回顾对象 | 数据复核、任务管理 | 字段结构、数据类型、规格限值、显示顺序 | 选择模版时 |
| 数据复核 | 数据表模版、回顾对象 | 数据分析 | 已复核确认的原始数据记录 | 分析执行时 |
| 分析模版 | 回顾对象 | 数据分析 | 分析类型、统计方法、图表类型、筛选条件 | 选择模版时 |
| 数据分析 | 分析模版、数据复核、回顾对象 | 分析结果 | 分析配置参数、数据源引用 | 分析执行时 |
| 分析结果 | 数据分析 | 回顾报告 | 统计指标值、控制图/趋势图、判定结论 | 报告编写时引用 |
| 报告模版 | 回顾对象 | 回顾报告 | 章节结构、占位符定义、内容要求 | 创建报告时 |
| **回顾报告** | 报告模版、分析结果、回顾计划 | 无（终端输出） | 报告内容、附件、审批记录 | 审批通过后 |
| **回顾计划** | 回顾对象 | 任务管理、回顾报告 | 计划基本信息、回顾对象、周期、负责人 | 审批通过后 |
| 任务管理 | 回顾计划、数据表模版、分析模版、报告模版 | 数据复核、数据分析、回顾报告 | 任务类型、关联配置、执行入口 | 计划批准后自动生成 |
| 首页 | 全部业务模块 | 无（聚合展示层） | 各模块的统计指标、待办事项、消息通知 | 定时刷新 + 实时推送 |

### 4.2 主线执行顺序

```text
   ┌──────────────┐
   │   回顾对象   │  ← 基础数据（CQA / CPP / IPC）
   └──────┬───────┘
          │ 引用参数
          ▼
   ┌──────────────┐         ┌──────────────┐
   │ 数据表模版   │         │ 分析模版     │
   └──────┬───────┘         └──────┬───────┘
          │ 实例化字段              │ 引用配置
          ▼                        │
   ┌──────────────┐                │
   │   数据复核   │  ← 原始数据   │
   └──────┬───────┘                │
          │ 提取已复核数据          │
          ▼                        ▼
          └────────► ┌──────────────┐
                     │   数据分析   │
                     └──────┬───────┘
                            │ 产出统计结果
                            ▼
                     ┌──────────────┐
                     │  分析结果    │
                     └──────┬───────┘
                            │ 引用插入
                            ▼
   ┌──────────────┐  实例化  ┌──────────────┐
   │  报告模版    │ ───────► │  回顾报告    │  ← 终端输出
   └──────────────┘         └──────┬───────┘
                                   │ 审批通过后回写状态
                                   ▼
                            ┌──────────────┐
                            │  回顾计划    │
                            └──────┬───────┘
                                   │ 已批准 → 自动分解
                                   ▼
                            ┌──────────────┐
                            │  任务管理    │  → 触发数据复核 / 数据分析 / 报告编写
                            └──────────────┘
```

## 五、数据流转关系

系统共定义 **10 条核心数据流**（DF-001 ~ DF-010），覆盖模块间的全部主数据交互。

| 数据流编号 | 发起方 | 接收方 | 流转数据内容 | 流转方式 | 触发条件 |
|---|---|---|---|---|---|
| **DF-001** | 回顾对象 | 数据表模版 | 对象参数列表（CQA/CPP/IPC 名称、规格限） | 引用关联 | 配置模版字段时选择参数 |
| **DF-002** | 数据表模版 | 数据复核 | 字段结构定义、规格限配置 | 模版实例化 | 选择数据表模版进行数据录入 |
| **DF-003** | 数据复核 | 数据分析 | 已复核数据记录集（状态=已复核） | 数据提取 | 执行数据分析时自动提取 |
| **DF-004** | 分析模版 | 数据分析 | 分析类型、统计方法、图表配置、最小样本量 | 引用关联 | 选择分析模版 |
| **DF-005** | 数据分析 | 分析结果 | 统计计算结果、生成的图表文件、判定结论 | 自动创建 | 分析执行成功完成 |
| **DF-006** | 分析结果 | 回顾报告 | 分析结果引用 ID、图表嵌入代码、统计摘要 | 手动引用 | 报告编写时插入 |
| **DF-007** | 报告模版 | 回顾报告 | 章节骨架、占位符列表、模版版本号 | 实例化生成 | 创建报告时选择模版 |
| **DF-008** | 回顾计划 | 任务管理 | 计划 ID、回顾对象、回顾周期、负责人信息 | 自动分解 | 计划状态变更为"已批准" |
| **DF-009** | 任务管理 | 数据复核 / 数据分析 / 回顾报告 | 任务 ID、任务类型、关联配置信息 | 导航跳转 | 用户点击"去执行"操作 |
| **DF-010** | 回顾报告 | 回顾计划 | 报告 ID、报告状态、生效时间 | 状态回写 | 报告审批通过后 |

### 5.1 数据流分类

- **引用关联型**（DF-001、DF-004）：上游数据被下游配置时引用，不复制数据本体。
- **实例化型**（DF-002、DF-007）：模版被实例化为具体业务记录，承载结构定义。
- **数据流转型**（DF-003、DF-005、DF-006）：业务数据在生命周期中流转，状态或形态发生变化。
- **任务驱动型**（DF-008、DF-009）：计划审批后自动分解为任务，任务驱动下游执行。
- **状态回写型**（DF-010）：终端输出反向影响上游计划状态，形成业务闭环。

## 八、生成用例的关键参考点

- **业务主线优先级**：基础数据层 > 数据管理层 > 分析管理层 > 报告管理层；跨模块联动用例应优先围绕主线流程展开。
- **数据完整性专项**：所有涉及数据录入 / 修改 / 复核 / 审批的模块必须覆盖 ALCOA+ 原则与审计追踪。
- **状态流转关键节点**：回顾计划「已批准」、数据复核「已复核」、分析结果「已完成」、回顾报告「已审批」。
- **跨模块联动重点**：DF-003（复核→分析）、DF-005（分析→结果）、DF-008（计划→任务）、DF-010（报告→计划）。
- **法规合规边界**：所有用例不得违反 GMP、ALCOA+、电子签名要求。
