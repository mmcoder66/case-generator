# CPV 界面设计图说明

## 存放内容

本目录用于存放 CPV 系统界面设计图、截图说明、交互说明、流程图、弹窗说明和视觉标注文档。

## 目录结构

UI 图按章节名（与 PRD 章节名保持一致）组织：

```text
inputs/cpv/ui_design/
  <章节名>/
    <页面 1>.png
    <页面 2>.png
  _incoming/
    README.md        # 收到图但还未确认归属时先放这里
```

- 目录名使用中文章节名，与 PRD 章节一致，AI 可直接按章节名定位
- 原始 Word 文件统一放在 `inputs/cpv/requirements/raw_docs/`，不放在本目录
- 新收到但未确认归属的图片先放入 `_incoming/`，确认后人工移入对应章节目录
- 目录可通过以下两种方式创建和维护：
  1. **自动提取**：运行 `python scripts/extract_docx.py --project cpv <docx> --section "<章节名>" --extract-images`，自动从 PRD Word 文档中提取指定章节下的嵌入图片
  2. **手动放置**：人工截图或导出图片后放入对应章节目录，可自定义图片名称

## 阅读重点

读取界面设计图时，应重点提取：

- 页面入口
- 页面标题
- 输入框、按钮、弹窗、提示条等控件
- 默认状态、选中状态、禁用状态和错误状态
- 列表、卡片、详情页、状态标签、进度条、图表等展示字段
- 菜单权限、按钮权限和无权限状态
- 表单字段、必填标识、字段类型、枚举项、长度限制和校验提示
- 审计追踪详情、操作历史、操作前后值
- 一键分析配置区域（横轴变量、纵轴变量、子组大小、分析方法选择）
- 控制图、帕累托图、拟合线图、箱线图等分析图表展示
- 参考线配置（新增、删除、拖拽、数据范围）
- 页面跳转关系
- 空状态、加载状态和失败状态

## 用例生成要求

界面设计图不仅用于验证页面展示，还应转化为交互测试用例。例如按钮是否可点击、无权限或状态不允许时是否禁用、错误提示是否展示、关键确认弹窗是否触发、页面刷新或返回后状态是否保留、跨模块跳转后数据是否一致。
