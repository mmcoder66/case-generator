# 导入功能专项覆盖规则

## 使用原则

涉及 Excel / CSV / 模板文件导入、批量导入、导入模板或错误报告时，必须先读取本文件，再生成或补充用例。

| 原则 | 要求                                                                            |
|---|-------------------------------------------------------------------------------|
| 事实来源 | 先按 PRD、模板字段表或用户补充说明确认字段名、必填性、条件必填、格式、长度和引用数据规则。                               |
| 职责边界 | 本文件只定义导入专项覆盖场景；优先级按 `priority_rules.md` 判定，难度按 `difficulty_level_rules.md` 判定。 |
| 生成范围 | 只生成与当前导入对象、模板字段和导入流程直接相关的场景，不扩展无关导入对象或无关字段。                                   |
| 信息缺失 | 若 PRD、模板或用户补充资料未说明导入策略、字段规则或错误报告格式，不得自行补充确定结论，应在需求覆盖率对照表中标记需要确认。              |

---

## 基础规则

| 场景 | 生成要求 |
|---|---|
| 字段原文 | 前置条件、步骤和预期中涉及模板字段时，必须使用 PRD / 模板原文，不得自行改名或概括。 |
| 表头批注 | 预期不得只写 `批注显示正确`、`说明完整` 等泛化描述，必须写明每个被校验字段的具体批注文案；模板字段较多时，按字段类别拆分多条用例。 |
| 下拉选项类字段 | 预期不得只写 `下拉选项正确`、`数据有效性正确` 等泛化描述，必须写明每个被校验字段的具体选项值或限制内容。 |
| 成功导入 | 导入支持必填字段和可选字段差异时，至少拆分 `只填必填项导入成功`、`填写所有字段导入成功`。 |
| 多行导入 | 需验证成功条数、列表刷新和字段落库；涉及用户、权限、引用对象或状态时，还需验证关联关系一致。 |
| 前置条件 | 不得写 `文件合格`、`字段完整`、`数据正常` 等模糊描述，也不得写步骤中才执行的动作。 |

## 异常与边界

按下表逐个字段 × 逐条适用规则核对导入覆盖情况，不得只按规则名扫一遍就跳过。快捷方法：参考同模块新增/编辑用例已有的字段校验，同一字段的同一校验在新增/编辑入口和导入入口都应有对应用例，导入侧缺失必须补齐或标注"不适用：原因"。

| 类型 | 覆盖要求 |
|---|---|
| 文件级异常 | 按当前导入入口的限制评估，覆盖文件类型、文件大小、空文件、无数据、无表头、表头不匹配、合并单元格、加密、损坏。 |
| 字段级异常 | 按模板字段逐项评估，覆盖必填为空、唯一性冲突、格式非法、长度边界、引用数据不存在、条件必填等适用规则。 |
| 长度边界 | 文本字段明确最大长度时，至少覆盖等于最大长度成功、超过最大长度失败。 |
| 多规则字段 | 同一字段同时包含格式规则和引用存在性规则时，格式错误和引用对象不存在必须拆成独立用例。 |
| 下拉字段 | 下拉字段由 Excel 模板数据有效性限制输入，不生成非法值导入失败用例；覆盖下拉选项是否完整正确、合法选项可保存，以及必填为空等实际可发生的导入校验。 |
| 条件字段 | 分别覆盖条件满足时的必填、格式、引用存在性，以及条件不满足时允许或不允许填写的取值规则。 |
| 多字段错误 | 同一行多个字段错误仅用于验证错误报告展示，不替代单字段独立校验。 |
| 全量回滚 | PRD 要求整份文件事务性导入时，必须验证任一行错误时整份文件拒绝导入，不产生部分导入脏数据。 |
| 部分成功 | PRD 支持部分成功时，必须验证成功行落库、失败行不落库、成功 / 失败数量和失败原因展示正确。 |

## 错误报告

涉及导入失败报告、TXT 错误报告或错误明细下载时，至少评估：

| 检查项 | 要求 |
|---|---|
| 错误定位 | 展示错误行号、字段名和失败原因。 |
| 多错误展示 | 同一行多个错误按约定格式展示。 |
| 下载一致性 | 错误报告下载后内容与页面失败原因一致。 |
| 数据状态 | 导入失败后列表数据不产生脏数据；涉及权限、引用关系或操作日志时，还需验证关联数据和日志记录不被错误写入。 |

## 命名与分组

| 场景 | 规则 |
|---|---|
| 导入失败类用例名称 | 必须写清具体失败原因，优先使用 `具体原因，导入失败` 或 `具体原因，上传失败`。 |
| 导入失败类分组 | 默认 `二级分组` 填写 `导入失败校验`，`三级分组` 按被校验字段或对象填写；如需进一步区分字段类别、错误报告或子功能，可增加 `四级分组`。 |
| 模板说明类分组 | 默认 `二级分组` 可填写 `模板下载`，`三级分组` 按字段类别填写；如字段类别仍不足以表达子功能，可增加更深分组列。 |
| 排序 | 相同优先级内，校验同一字段或同一类导入对象的用例必须相邻放置。 |
