# 优先级规则

## 优先级取值

测试用例优先级只能使用：

- `P0`：阻塞性，必须在发布前通过。只用于核心业务正向主链路、跨模块闭环成功和核心结论准确性。
- `P1`：重要，应在版本内完成。用于重要分支、权限访问控制、常用操作、关键配置正常维护和中等风险回归。
- `P2`：一般，有时间时覆盖。用于字段校验、异常提示、边界值、展示体验、兼容适配、弱网和防重复类稳定性。

---

## P0 规则

满足以下**任一**条件，标记为 `P0`：

- 核心业务链路的正向主流程。
- 跨模块闭环的成功验证（上游动作触发下游动作按预期联动）。
- 导入成功场景中，对核心业务数据写入、关键配置生效、权限生效、跨模块数据可用性或后续核心流程有发布阻断影响的主链路。
- 导出成功场景中，导出文件承载核心业务数据或关键配置，且导出结果会影响后续复用、归档、迁移或业务核对的主链路。
- 主链路成功后，核心计算结果、关键结论、引用结果必须正确的场景。
- 高频核心入口的正常成功链路。
- 明确属于发布阻断项的核心追溯验证：PRD 指定必须记录的关键操作成功后，日志记录、详情基本信息和内容详情字段完整且准确。

## P1 规则

满足以下**任一**条件，标记为 `P1`：

- 重要分支流程，如审批退回、重新提交、修订、失效、升版、关闭、重新生成。
- 影响用户继续完成常用业务操作的重要异常处理，如账号密码错误、无按钮权限或关键数据不可用。
- 未登录、登录态过期、无菜单/按钮/数据权限、跨租户 / 跨组织 / 跨数据域访问或接口越权后仍可执行敏感操作。
- 高频查询、筛选、排序、分页、批量选择、导入导出、状态筛选、待办入口等日常操作；其中导入成功后影响核心业务数据、关键配置、权限生效或跨模块闭环的主链路按 `P0`，导出结果影响后续复用、归档、迁移或业务核对的主链路按 `P0`。
- 重要状态变化的页面展示和操作范围（具体状态名称按新项目业务填写）。
- 阻断常用业务流程的系统级异常或关键数据不可用；单一功能的数据异常拦截、边界处理、提示展示按 `P2`。
- 已知的中等风险操作路径或常见历史问题的回归验证。
- **业务文案模板验证**：按触发条件的紧急程度判定。待办文案中需及时处理（审批、退回、超期、逾期、待复核）的标 `P1`，提醒类（即将到期、待启动、转派）的标 `P2`；消息通知文案一般为 `P2`（通知知晓类）。

## P2 规则

满足以下**任一**条件，标记为 `P2`：

- 字段、文件和配置项校验场景，如必填校验、字符长度校验、字段格式校验、数据类型限制、默认值、禁用态、范围提示、非法文件格式或内容、表头不匹配、文件大小限制等。
- 业务异常拦截和边界处理，如样本量不足、数据量不一致、数据值全相同、缺失值静默处理等单一功能异常场景。
- 低频边界场景，如超长名称截断、极大列表展示、备注超长、特殊字符展示、日期控件边界显示。
- 文案、样式、布局类问题，如字号、颜色、间距、对齐、空状态插图、提示文案轻微不一致。
- 模板说明和提示文案场景，如模板字段批注、下拉选项说明、错误提示文案、帮助说明等。注意：业务文案模板（如待办文案、消息通知文案）不归入此类，按 P1 规则中的"业务文案模板验证"判定。
- 弱网、重复点击和防重复提交类稳定性场景，如重复点击导致重复提交、重复审批、重复生成任务、重复导入、重复导出等。
- 不影响核心业务判断的**辅助功能**，如最近访问记录、默认展开状态、列宽调整、表格列固定、列表排序记忆。
- 纯兼容或展示适配场景，如不同浏览器、不同分辨率、缩放比例、横屏、打印预览样式。
- 不影响核心业务状态的体验优化，如加载动画、骨架屏、按钮 hover 状态。
- 低风险操作路径、低频配置项、说明性内容或帮助入口验证。

## 优先级判断原则

无法直接判断时，按以下顺序评估：

1. 是否为核心正向主链路或跨模块闭环成功验证？是 → `P0`。
2. 是否直接验证关键结果或引用结果的核心正确性？是 → `P0`。
3. 是否为 PRD 明确要求的关键追溯记录或详情字段准确性验证，且会影响发布合规性？是 → `P0`。
4. 是否为权限访问控制、重要分支流程、常用操作或正常关键配置维护？是 → `P1`。
5. 是否为字段校验、配置异常、提示文案、边界值、兼容展示、弱网或重复点击稳定性？是 → `P2`。
6. 同一场景同时满足多个等级时，按最高业务影响取值；但本文件已明确归入 `P2` 的异常提示、边界和展示类场景不再上提。

## 优先级分布建议

一个模块的完整用例集，优先级分布参考如下：

| 优先级 | 建议占比 | 说明 |
|---|---|---|
| P0 | 15%–20% | 聚焦核心主链路、跨模块闭环、结论准确性和发布阻断项，不应过多 |
| P1 | 35%–50% | 覆盖重要分支、权限访问控制、常用操作、正常关键配置和中等风险回归 |
| P2 | 30%–45% | 覆盖字段校验、异常提示、边界、兼容、样式、展示和稳定性 |

若 P0 用例明显超过 20%，说明优先级标注可能过于保守，应重新评估；若核心正向主链路、跨模块闭环或核心结果正确性场景未标为 P0，应优先修正。
